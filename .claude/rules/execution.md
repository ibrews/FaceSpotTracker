---
description: Core execution discipline rules — apply to all tasks on all machines
---

- **Surface ambiguity before acting.** If a request has multiple valid interpretations, name them and ask — don't silently pick one. If something is unclear, stop and say what's confusing rather than assuming.
- **Orphan ownership on cleanup.** Remove dead code, imports, and variables that *your* changes made unused. For pre-existing dead code you notice, mention it — don't delete it without being asked.
- **Numbered plan + verify for multi-step tasks.** Before starting anything with 3+ steps, state a numbered plan with a verification step for each. This creates checkpoints and avoids silent drift.
