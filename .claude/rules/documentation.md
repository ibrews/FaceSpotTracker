---
description: Documentation discipline — after any feature change, update project docs
---

After adding or changing features in any project:
- **Update the README** if it exists — add new features, update screenshots/tables, keep examples current. Don't wait to be asked.
- **Update HANDOFF_PROMPT.md** if it exists — keep the AI handoff accurate so future sessions aren't stale.
- **Write to the KB** when a decision, pattern, or strategy emerges that could be useful later (`intelligence/decisions/` or the relevant department folder).
- **Update CLAUDE.md / .claude/CLAUDE.md** in the project if the change affects how an AI should work with the codebase.

Rule of thumb: if you'd have to explain the change to someone who opens the project tomorrow, write it down now.
